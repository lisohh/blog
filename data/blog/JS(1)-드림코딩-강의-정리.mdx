## 1. 자바스크립트의 역사(1강)

> 새로운 언어를 배울 때는 바로 문법에 전념하는 것보다<br />
> 그 언어의 탄생배경에 대해 이해하는 것이 중요하다

- 어떻게 탄생했으며
- 어떤 문제를 해결하기 위해 만들어졌으며
- 이 언어를 연마했을 때 어떤 분야에 더 활용할 수 있는지, 이익은 무엇인지
  를 이해해야 공부방법이 명확해진다.

```css
1.  마크 앤더슨이 정적인 웹사이트에서 좀 더 동적인 웹사이트를 만들기 위해
    Scripting 언어를 추가하기로 결심

2.  브랜든 아이크의 스킴 스크립트 언어의 컨셉을 가져오되 문법은 자바스타일로
    스크립트 언어를 만들기로 했고 1994년에 라이브스크립트 언어가 출시됨.
    라이브스크립트의 인기를 끌기위해 자바스크립트로 이름을 변경해서 1995년 출시.
    마이크로소프트사가 넷스케이프의 자바스크립트를 Reverse Engineering해서
    코드를 고대로 베껴와 Jscript를 출시함.

3.  두 언어가 경쟁을 하다가 참다못한 넷스케이프는 ECMA를 찾아가
    1997년 첫번째 ECMAScript 언어 정의 문서가 나왔고
    그뒤로 계속 버전이 업데이트 되고 있음
    그래서 마이크로소프트는 이 표준화에 맞추지 않고 떨어져나오게 되는데
    2004년 모질라의 Firefox가 ECMA에 맞춰 다시 표준화를 시도하게 된다.

4.  2004년 제시 제임스가 AJAX(비동기 프로그래밍)을 내놓고
    시장에 다른 브라우저들도 나오게 되면서
    웹 시장이 커지고, 개발자 커뮤니티가 생겨나며 점점 커지게 된다.
    이들을 위한 API 라이브러리(jQuery, dojo,mooTools)같은게 나오게 되면서
    개발자들은 표준화 걱정없이 API를 통해 개발을 할 수 있게되었다.

5.  2008년 구글에서 크롬이라는 브라우저를 만들게 되면서
    JIT(just-in-time compilation)라는 정말 강력한 엔진이 포함된
    브라우저를 내놓음 -> 다른 브라우저들이 위기의식을 느낌
    그래서 드디어 2008년 7월에 여러 브라우저가 뭉쳐 표준화를 의논하게 됨.
    2009년 ECMAScript5 / 2015년 ECMAScript6가 나옴

6.  자바스크립트는 이제 잘 정착된 언어로 다른 브라우저도 표준화를 잘 따라가므로
    이제 jQuery같은 라이브러리 없이 바로 API를 통해
    모든 브라우저에서 잘 동작하는 웹을 개발할 수 있게 되었다.

7.  브라우저에는 다양한 자바스크립트 엔진들이 있는데
    V8을 크롬이 내놓으면서 node와 ELECTRON 및 마이크로소프트가 이 엔진을 이용한다.

8.  최신버전의 ECMAScript6를 사용해서 개발후 Babel 컴파일러를 통해 변환한다.
```

### 요즘의 동향

**SPA(Single Page Application)**<br />

하나의 페이지 안에서 데이터를 받아와서 필요한 부분만 부분적으로 업데이트하는 애플리케이션
이를 더 쉽게 하기 위해 React,Vue,Angular,Backbone같은 라이브러리,프레임워크들이 나옴

**node.js**<br />

V8 자바스크립트 엔진을 이용해 백엔드에서 서비스를 구현할 수 있도록 만든 도구

**WA(Web Assembly)**<br />

브라우저에서 동작할 수 있는 언어는 자바스크립트가 유일했는데 요즘엔 WA라는 언어가 나와서 Rust,C,C# 등을
통해 웹어플리케이션을 만드는게 가능해졌다.
