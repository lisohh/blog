# 코딩 공부 하는 법

오늘은 내 자신을 위해 코딩 공부하는 방법을 영어 공부하는 법과 비교해서 써보려고 한다.<br/>
둘 다 언어를 공부한다는 공통점이 있기 때문에 내가 영어를 공부했던 방법을 코딩 공부에 적용해보면 <br/>
좋을 것 같다는 생각이 들었다.

## 개발 용어 외우기 = 영어 단어 외우기

#### 1. 어원 알기<br/>

영어 단어를 외울 때 어원을 알면 비슷한 단어를 배울 때 의미를 알거나 발음을 알기가 편하다. <br/>
개발 용어도 출처와 원형을 알면 거기서 파생된 용어를 배울 때 이로울 것이라고 생각한다.<br/>
비전공자인 나는 CS 지식이 부족하기 때문에 구글링과 책을 통해서 찾아보도록 해야지.<br/>

#### 2. 반복 학습하기<br/>

영어 단어도 여러번 암기하고 잊고 또 암기해야 오래 기억에 남았다.<br/>
중요한건 암기를 하기는 해야 한다는 것이다. <br/>
개발 용어도 그 때 그 때 뜻을 이해하고 암기를 일단 한다음 까먹은 후에 또 외우는 식으로 하려고 해야겠다.

#### 3. 응용 형태도 함께 학습하기<br/>

단어를 알 때 품사와 과거형,과거분사형 등 여러 형태를 함께 학습하고 어떻게 쓰이는지 문장을 통해 확인하면<br/>
훨씬 더 자연스럽게 단어의 쓰임을 익힐 수 있다. <br/>
개발 용어 또한 공부 할 때 공식 문서나 책, 인터넷 서핑을 통해 응용형을 확인하고 <br/>
쉬운 형태의 코드를 보며 사용을 익히면 좋겠다.

## 문법 익히기

#### 1. 기본 형식 공부하기<br/>

영어를 익힐 때 기본 5형식 문장을 기본적으로 익혔다.<br/>
먼저 어떤 구조로 이루어졌는지 읽고 써보고, 예시 문장을 많이 보고 써보았다. <br/>
프론트엔드를 배우는 나로서는 기본 html,css,js의 구조를 많이 보고 익히는게 기초를 탄탄히 하는데 도움이 된다.<br/>
기본을 빨리 배우고 넘어가는 사람들도 있지만 나는 기본을 언제든 다시 보면서 계속해서 익히는게 도움이 많이 됐다.

#### 2. 문장 많이 보기<br/>

책이나 영상, 음악으로 응용된 문장들을 많이 접하면 자연스럽게 문법을 체화할 수 있다.<br/>
물론 아주 쏙쏙 다 흡수되는 것은 아니지만, 자연스럽게 어떤 단어는 어떤 식으로 써야하지? 하고 감이 생긴다.<br/>
이 때도 어렵고 복잡한 문장이 아니라 쉬운 문장부터 보고 듣는게 좋아서<br/>
코딩도 단순한 코드부터 많이 접해야겠다.<br/>

#### 3. 짧은 글 써보기<br/>

일기든 편지든 채팅이든 짧은 글이라도 자꾸 써봐야 실제로 문장을 만드는 실력이 늘고 어휘도 더 많이 사용하게 된다.<br/>
코딩 또한 긴 프로젝트가 아니더라도 짧은 글을 자꾸 써보려고 해야겠다.<br/>
정말 짧은 코드라도..!

## 긴 글 쓰기

#### 컨셉(기획) - 폴더 구조 잡기(백, 프론트 흐름 이해) - 짧은 글 쓰기(작은 기능 구현) - 글 연결(기능끼리 연결)

여기까지는 아직 내가 도달하지 못했지만, 영어 공부로 비교하여 일단 써놓고 나에게 적용해보아야겠다.<br/>
영어도 긴 글을 쓰려면 한글로 글을 쓰듯 일단 컨셉을 잡고, 구조를 잡고, 구조에 맞게 짧은 글들을 써서 연결을 시키고 다듬어야 한다. <br/>
코딩을 공부하면서는 코드를 구조로 잡는 일부터 어려운데, 이건 글과는 다르게 폴더 구조와 백엔드, 프론트엔드의 연결 흐름을 익혀야 하기 <br/>
때문인 것 같다.
그럼 코딩에 맞춰서는 <br/>
컨셉(기획) - 폴더 구조 잡기(백, 프론트 흐름 이해) - 짧은 글 쓰기(작은 기능 구현) - 글 연결(기능끼리 연결) 하는 순서로 <br/>
진행해야겠다.

## 이 모든 걸 병렬적으로

중요한 건 단어,문법,짧은 글쓰기,긴 글쓰기를 한꺼번에 병행하며 학습하는 것이다.<br/>
이 부분이 어려운데, 시간을 잘 분배하기도 어렵고, 한 부분에서 막히면 <br/>
짧은 글쓰기나 긴 글쓰기같은 실제 응용 부분에서 진도가 영 안나가기 때문이다. <br/>
하지만 이럴 때 내가 단어나 문법을 완벽히 모른다고 해서 글쓰기를 멈추면 안된다. <br/>
최대한 짧은 글쓰기라도 꾸준히 해나가면서 주변에 피드백을 구하고 고쳐나가야 <br/>
긴 글을 쓸 수 있다. <br/>
지금 나는 단어와 문법을 어느정도 공부하면서 짧은 글쓰기를 연습하는 단계라고 할 수 있다. <br/>
글쓰기를 멈추고 싶을 때가 많지만 하루에 한 문단이라도 쓰려고 노력해보면 <br/>
영어를 공부하듯이 코딩도 조금씩 늘겠지?<br/>

여기까지가 내가 영어를 공부했던 방법을 코딩에 적용해서 써본 것이다.
머릿속에만 존재하던 생각을 풀어볼겸, 그리고 나처럼 공부하는 분들이 있다면 <br/>
이런 방법도 있다고 알려드리고 싶어서 글을 써보았다.<br/>
나 자신도 이 방법을 토대로 연말까지 긴 글(프로젝트)을 두 세번은 써볼 수 있으면 좋겠다.
